CTFHub-Web-RCE

​ RCE(Remote Command/Code Execute),远程命令执行ping和远程代码执行evel。一般出现在常见的网络设备，防火墙，路由器等web管理界面。一般是会给用户一个可操作的web界面，来执行命令，返回结果。由于存在过滤不充分等原因，导致后台被控制，这就是rce漏洞。

我们直接试着直接利用system()访问目录文件

过滤cat

题目：过滤了cat命令之后，你还有什么方法能读到 Flag?

​ 我们在cat被过滤后，替换cat有几个常见的命令

  • tac：与cat相反，cat是从头到尾显示，tac从尾到头
  • more：根据窗口大小，一页一页的现实文件内容
  • less：和more类似，但其优点可以往前翻页，而且进行可以搜索字符
  • head：只显示头几行
  • tail：只显示最后几行

​ 这题算是上一题的升级了，我们一样可以用ls来读取文件目录

由于源码中if (!preg_match_all("/cat/", $ip, $m))对cat进行了替换，禁用了cat，我们可以用tac来读取文件。

过滤目录分隔符

题目：这次过滤了目录分割符 / ，你能读到 flag 目录下的 flag 文件吗

if (!preg_match_all("/\//", $ip, $m))进行过滤目录分割符。

很明显这里是一个文件夹，但是我们分隔符被过滤了进不去，因此用cd进行目录跳转。

发现了flag目录，直接进行读取了，源码中进行读取

综合过滤练习

题目：同时过滤了前面几个小节的内容, 如何打出漂亮的组合拳呢?

if (!preg_match_all("/(\||&|;| |\/|cat|flag|ctfhub)/", $ip, $m))过滤的东西比之前的多了，但是问题不大。

|

&

;

空格：<

/

cat

flag

ctfhub

我们直接在url位置进行注入将;换成%0a

flag_is_here可以利用$(printf "路径")代替路径，而路径文字可以利用十六进制字符串进行替换。空格则使用%09进行绕过。

cat用tac绕过，路径同样用十六进制编码绕过。
